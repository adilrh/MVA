I) Openfst / Pywrapfst¶

The OpenFst library is a C++ template library. It is a library for constructing, combining, optimizing, and searching weighted finite-state transducers (FSTs) The C++ code is used in many speech applications, and has been developped by NYU Courant Institute and Google Research.

A link to the C++ api and the possible operations http://www.openfst.org/twiki/bin/view/FST/FstQuickTour#CreatingFsts

Exercice 1) Basic operations on Finite State Transducers (FSTs)¶

a) Build a finite state transducer that converts a sequence in Upercase to Lowercase with input vocubaluary $\Sigma = \{A,B\}$, and output vocabulary $\Omega = \{a,b\}$

b) Now, you want to to be able to handle, in addition to $\Sigma = \{A,B\}$, extra symbols in your input sequence such as $\{@,a,b,!,1,2,\_,-,\}$ how to modify the precedent transducer to do so?

c) Use the Union operation to answer the previous question.

d) You do not want to support anymore the numeric characters, $ \{1,2\}$, remove them with the Difference operation between two chosen FSTs.

e) Now, you want to go directly from $\Sigma = \{A,B\}$ to $\Delta = \{\alpha,\beta\}$, use the Composition operation, a previous built FST a new one to have such a FST.

Exercice 2) Edit Distance and Weighted Finite State Transducers (WFSTs)¶

It is possible to compute the usual Levenshtein distance or Edit Distance with Weighted Finite State Transducers. In the cell below, it is an example of the operations to compute the Edit Distance with only additions taken into account for basic sequences from this Vocabulary $\Sigma = \{0,1\}$. You use the composition twice: $ S_1 \circ EditFST \circ S_2$. You obtain the Graph FST of all possible paths to go from one sequence to the other. Then you use the shortest path to obtain the Edit Distance.

a) In this part the vocabulary of the Language is $\Sigma = \{h,o,m,l,a,c, g\}$. Build a transducer for the Edit Distance with the additions, substitutions and deletions. You should use a small code snippet to not type everything by hands. Hint: Introduce some intermediate transitions type. Provide an explanation of the resulting weights and input/output transitions.

Any comments on the occupied memory for the WFST and $|\Sigma|$? Do you have a solution to reduce it? You are solving a shortest-path problem in a graph with non-negative weights to compute the finale edit distance. In the case of Edit Distance in linguistics, which algorithm should you use ? Justify your choice.

b) Now, you are provided with a dictionnary of correct words. You want to detect the misspelled words in a text. The naive way without FST is to check whether or not each word of a text is present in the words vocabulary. Another way is: 0) Build a FST of your input text where each word is represented by a path 1) Compose your text FST with the dictionnary FST. You obtain only the intersection of the input words and dictonnary words. 2) Use the difference operation between the obtained FST and the input FST to obtain the mispelled words. Here is a FST with correct spelled words. Could you show some text inputs and use the previously described operation to detect misspelled ones.

Your vocabulary for this question is dict = {'coml','almanach'}, the text with mispells is T = 'cogml coml almanach lomh comll lamanach cocoml'.

c) How can you use the Edit Distance WFST and the dictionnary to correct mispelled words?

d) Bonus - confusion matrix : When you are typing on your Qwerty keyboard you are less likely to make the mistake $s \rightarrow p$ than $s \rightarrow d $. When you use a recognizer of tri-phones, the algorithm is less likely to make non-valid phonotactically mistakes of never seen sequences. Imagine you are provided a confusion matrix with different penalties weights, how can you adapt your WFST in consequences?

def add_arc(alphabet,add_fst,fst):
    for i in range(1,len(alphabet)):
        add_fst.add_arc(s, fst.Arc(i, i, 0.0, s))
    for i in range(1,len(alphabet)):
        add_fst.add_arc(s, fst.Arc(0, i, 1.0, s))
    for i in range(1,len(alphabet)):
        add_fst.add_arc(s, fst.Arc(i, 0, 1.0, s))
    for i in range(1,len(alphabet)):
        for j in range(1,len(alphabet)):
            if alphabet[i]!=alphabet[j]:
                add_fst.add_arc(s, fst.Arc(i, j, 1.0, s))

The add_arc method handle the 3 possible operation, if one operation is occurred a weight of 1 is assigned as a penality, 0 otherwise: Unedited character ($h:h/0$), inserted character (i.e $\epsilon:h/1$), deleted character (i.e $h:\epsilon/1$) and substitued character (i.e $h:c/1$) with respect to the edit distance between words

The spatial complexity (occupied memory) of the WFST is according to the add_arc method that I wrote O($|\Sigma|^2$), but it is not the perfect method, we could reduce the spatial complexity by using hash tables and then storing only the keys of the hashs not the chars.

Since the weights are positive, Djikstra is the suitable algorithm to find the shortest path from a deparature node to an arrival one (it supports cycles). It's the best one in term of complexity $O(a +n log n)$ where a is the number of arcs and n the one of nodes.

c) There are 4 acoustic models:

Monophone GMM: that models each level (i.e phone) by a GM distribution independently, the parameters to learn are the mean-vectors and the covariance matrices, this doesn't model the problem efficiently, however its complexity is the lowest among the other models so it's fast according to time-consuming.

Independent GMM/HMM: which consider that each level has a HMM/GMM modelisation and then there are more parameters to learn: the transition kernels between the hidden states which is in the discrete case a matrix $K$x$K$ where K is the number of possible values for each hidden state, also the parameters of the initial state distribution should be estimated and the GMM parameters for each observed state. We use MLE (EM algorithm) to estimate those parameters. However it consider that the phones are independent and then the covariance matrices are diagonal, however this is not always true since the mfbank features aren't independent for instance and this model doesn't work efficiently on them. Moreover, it's time-consuming than the previous method but more efficient

The adapted GMM/HMM which model the language and uses a MAP instead of a MLE and then models better the corpus however it's more time-consuming since more parameters are added, it's efficient when the training data is not sufficient (it's known that the MAP estimates tends in infinity amount of indepedent observations to MLE).

The more efficient method is HMM- DNN which uses a DNN to estimate or learn the parameters of the HMM and it's the best acoustic method practically however it's complexity increases dramatically with the size of the DNN and it's time consuming.

e) L.fst is the lexicon finite state transducer
f) The ASR system could be summarized in 3 steps:

-Feature extraction: using mfbank or ideally mfcc that gives 39 dimensionnal features that are independant.

-Acoustic $P(O/Q)$ , prononciation $P(Q/W)$ and language $P(W)$ (n-gram) modeling: By using the previous models and calculate the best parameters that fit the model either by MLE or MAP.

-The viterbi Algorithm: Dynammic Programming to calculate the most likely sequence of states
